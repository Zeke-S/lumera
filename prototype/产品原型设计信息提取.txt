你可以直接根据以下五个维度进行原型构图：
一、 产品核心定位与交互哲学
	•	产品名称：曜境 Lumera。
	•	核心灵魂：文理双核。不仅是像素的堆叠（感性/叙事），更是真理的演化（理性/逻辑）。
	•	交互范式：从“线性视频”转向**“多分枝交互知识状态机”**。
	•	核心原则：
	•	语义对齐：画面上的每一处跳动必须有物理/数学逻辑支撑（LKM-P 协议）。
	•	呼吸感/物理感：拒绝死板的线性移动，所有 UI 变换应具备弹簧（Spring）物理反馈。
二、 关键 UI 场景与组件 (Prototyping Elements)
1. 核心交互中枢：Omnibar (全能输入框)
	•	外观：位于屏幕底部的悬浮长条，极简设计，类似命令面板。
	•	功能：意图输入（自然语言）、参数实时修改、Agent 状态反馈。
	•	原型细节：输入时，下方应实时浮现 “逻辑树缩影”，展示 Agent 正在生成的 LKM-P 节点。
2. 三管线融合画布 (The Canvas)
	•	叙事层 (Background)：高质量视频背景（如雨天、实验室、宇宙背景）。
	•	逻辑层 (Overlays)：浮动在背景上的 SVG 矢量公式、数据卡片、动态推导线条。
	•	空间层 (3D Objects)：可被交互操作的 3D 模型热区（如千斤顶、mRNA 分子）。
3. 分枝导航器 (Branching Map)
	•	形态：类似地铁路线图或思维导图的可视化面板，隐藏在侧边或底部。
	•	功能：展示当前场景的所有“知识路径”，用户可以直观看到“如果我选了错误操作，视频会跳到哪个分枝”。
4. 时空透视滑块 (Perspective Slider)
	•	功能：不同于普通进度条。左右滑动时，不仅是时间的前进，更是**逻辑深度（Logic Weight）和叙事氛围（Narrative Weight）**的实时占比切换。
三、 核心业务流 (User Journey)
在原型中，你需要体现这 7 个步骤的连贯性：
	•	意图识别：用户输入“我想看勾股定理的动态证明”。
	•	事实锚点：系统调取公式 $a^2 + b^2 = c^2$ 锁定为真理底座。
	•	认知策略：Agent 决定采用“面积切割生长法”。
	•	场景描述：定义环境（黑板风格或 3D 抽象空间）。
	•	分镜描述：拆解为“公式出现、面积生长、数值对齐”三个逻辑分镜。
	•	分镜生成：调用 Remotion 云端渲染逻辑视频块。
	•	分镜组装：形成可点击跳转的互动成品。
四、 渲染管线技术特征 (用于视觉表现)
	•	逻辑管线 (Logic)：
	•	SVG 路径生长：图形不是突显，而是从线段“推”成面。
	•	公式矢量化：字符飞入、变形、高亮。
	•	叙事管线 (Narrative)：提供景深、光影、氛围。
	•	空间管线 (Spatial)：提供多角度透视和 3D 热区反馈。
五、 底层协议与架构逻辑 (支撑后台原型)
	•	LKM-P 协议：它是所有分镜的“说明书”，记录了每个时间点的变量状态。
	•	单 Agent + 多 Skills：
	•	Skill 1: 事实溯源 (Truth Retrieval)
	•	Skill 2: 认知编译器 (Cognitive Compiler)
	•	Skill 3: 视频导演 (Video Architect)
	•	双网关设计：
	•	MCP 网关：连接本地专业数据库、3D 资产库。
	•	API 网关：连接外部大模型（Sora、Veo）和云端渲染集群。
💡 绘制原型的顶级建议：
	•	突出“生长性”：在原型草图中，用箭头标注组件的演化方向，强调“从一个点长出一个正方形”的过程。
	•	体现“层级感”：明确标注哪些是“像素背景层”，哪些是“矢量逻辑层”。
	•	交互反馈：在原型中画出“当用户点击公式中的 $a^2$ 时，画面中对应的直角边闪烁”这种跨模态联动。
